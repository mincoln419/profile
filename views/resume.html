공영쇼핑 영업시스템 및 기초 데이터베이스 유지보수
유지보수 인력이 유지보수와 동시에 내부적으로 개발 프로젝트 진행하는 업무구조

담당업무
물류파트
* 유지보수업무 - 출고/회수, 반출, 입고/재고 주담당
- 물류업무 특성상 자동배치프로그램과 택배사, EMS, LMS, 카카오톡 등 인터페이스가 많음
- 배치프로그램은 오라클 Procedure 기반의 배치와 스프링부트 배치가 혼재하고, 특히 Procedure가 많아서 PL/SQL 작업에 많은 시간을 할당
- 현업 요구사항에 따라 인덱스 추가가 많은 편인데, 이때문에 기존 플랜이 바뀌어서 속도 저하가 발생하는 경우가 많아 새로운 인덱스를 연구하거나 힌트를 추가하는 경우가 많았음.
- 데이터량이 많은 주문, 상품 테이블과 물류 테이블 다수를 join 하다보니 병렬 처리로 속도개선한 케이스도 있음
- 업무를 담당하는 동안 지속적으로 모니터링 하며 SQL 튜닝작업을 한 결과 100개 되었던 timeout APM Exception이 철수할때쯤에는 5건 이하로 떨어짐
* 프로젝트성 개발 건
- 공급계획 출고지연패널티 적용 프로그램
- 출고지시 제외 설정 프로그램
- 직매입 품의 - 그룹웨어 - 전자계약시스템 연동 : 2020.06. ~ 2020.09.
  그룹웨어 업체: 더존/ 전자계약시스템 업체: 아이서트
  연동방법: 그룹웨어 - API Interface / 전자계약 - DB to DB

우정사업본부 차세대 - 프레임워크 전환 프로젝트

2019.01. ~ 2019.11.
- 수행사: LG CNS
- 고객사: 우정사업본부
내용
인공지능형 우편관리시스템 개발 사업 중 프레임워크 전환 프로젝트
Struts 기반의 구 프레임워크와 가우스로 개발된 업무시스템을 Spring기반의 DevOn 프레임워크에서 구동되도록 프레임워크 전환, 일부 메뉴 exbuilder6 로 리뉴얼 나주 개발팀의 exbuilder6 신규화면 연동테스트 지원
개발환경
  • Java Project
  • OS: UNIX
  • WAS: ZEUS
  • Web Aplication: WebtoB
  • Framwork: DevON(LG CNS), exbuilder6(토마토시스템), mybatis
  • DB: Oracle
  • DB tool: SQL Developer
  • Deploy: Jenkins
담당업무
프레임워크 전환팀.
1차: 실시간 우편현황 모니터링 파트, 우편장비관리, 우편통계 등 프레임워크 전환 담당
2차: PDA 담당 직원의 갑작스러운 철수로 PDA 인터페이스 전환작업 및 테스트 담당
1차 전환작업에서는 특이사항 없이 일정에 앞서 전환작업 및 테스트, 통합테스트 지원까지 정상 수행
2차 전환작업에서 인수인계후 프로그램 분석 중, PDA 담당 직원가 테스트 없이 기계적인 전환작업을 한 것이 확인
혹시나 했지만 역시나 인터페이스 통합테스트에서 결함이 발견
가장 큰 오류의 원인은 데이터 조회 순서대로 JSON으로 된 전문이 생성되어 전달되어야 하는데 단순 HashMap으로 인터페이스 모듈에 설정되어있는 것이 원인
오류원인 확인 후 LinkedMap으로 변경하는 작업을 하고, 모든 PDA 세부 모듈에 대한 다른 오류 확인 작업과 수정을 해야한다고 판단.
신속하게 PL과 PM에게 보고후, 회의를 통해 추가 인원 투입에 대해 결정 -> 2개월 남짓한 기간동안 인터페이스 전환 작업 및 통합테스트까지 수행 완료
